# Setting Up Your LangChain Dev Environment

With the core concepts of LangChain understood, it's time to get our hands dirty. A proper development setup is the foundation of any successful project. It ensures consistency, prevents dependency conflicts, and keeps your secret keys safe.

This guide will walk you through creating a professional, isolated Python environment for all your future LangChain development.

## Prerequisites

Before we begin, make sure you have the following installed:
- **Python 3.8 or newer:** You can check your version by running `python --version` or `python3 --version`.
- **A code editor:** We recommend [Visual Studio Code](https://code.visualstudio.com/) with the official Python extension.

## Step 2: Create and Activate a Virtual Environment

A virtual environment is a self-contained directory that houses a specific Python installation and its packages. This is a critical best practice to avoid "dependency hell."

<div class="flex items-start bg-gray-50 dark:bg-gray-800 p-4 rounded-lg my-4">
  <Terminal class="w-5 h-5 text-gray-600 dark:text-gray-400 mt-0.5 mr-3 flex-shrink-0" />
  <div>
    <h4 class="font-semibold text-gray-900 dark:text-white mb-1">Create the Environment</h4>
    <p class="text-gray-600 dark:text-gray-400 text-sm mb-2">
      This command creates a sub-directory named `.venv` containing the Python interpreter and libraries.
    </p>
    <code class="text-sm text-gray-800 dark:text-gray-200">
      python -m venv .venv
    </code>
  </div>
</div>

Now, activate the environment. The command differs based on your operating system:

**On macOS / Linux:**
```bash
source .venv/bin/activate
```

**On Windows (Command Prompt):**
```bash
.venv\Scripts\activate
```

**On Windows (PowerShell):**
```bash
.venv\Scripts\Activate.ps1
```

## Step 3: Install LangChain Libraries

With our environment active, we can now install packages. They will be installed *only* inside `.venv`, not globally.

We'll start with the essential packages:
- `langchain`: The core package that brings everything together.
- `langchain-openai`: The partner package for interacting with OpenAI's models.
- `python-dotenv`: A handy utility for managing environment variables (like API keys).

```bash
pip install langchain langchain-openai python-dotenv
```

## Step 4: Securely Manage Your API Keys

<div class="alert-warning">
  <div class="flex items-start">
    <ShieldCheck class="w-5 h-5 text-amber-600 dark:text-amber-400 mt-0.5 mr-3 flex-shrink-0" />
    <div>
      <h4 class="font-semibold text-amber-900 dark:text-amber-100 mb-1">Security Best Practice</h4>
      <p class="text-amber-800 dark:text-amber-200 text-sm">
        **Never** hard-code your API keys directly in your Python scripts. This is a major security risk. We will use a `.env` file to store them securely.
      </p>
    </div>
  </div>
</div>

1.  **Create a `.env` file** in the root of your `langchain-journey` directory.
    ```bash
    # On macOS/Linux
    touch .env

    # On Windows
    echo. > .env
    ```

2.  **Add your OpenAI API key** to this file. Get your key from the [OpenAI Platform](https://platform.openai.com/api-keys).
    ```text
    # Inside your .env file
    OPENAI_API_KEY="sk-YourSecretKeyGoesHere"
    ```

3.  **Create a `.gitignore` file** to ensure you never accidentally commit your secrets to version control.
    ```bash
    # On macOS/Linux
    touch .gitignore

    # On Windows
    echo. > .gitignore
    ```

4.  **Add `.env` and `.venv` to your `.gitignore` file**:
    ```text
    # Inside your .gitignore file
    .venv
    .env
    __pycache__/
    *.pyc
    ```

## Step 5: Verify Your Setup

Let's write a tiny script to confirm everything is working. Create a file named `verify_setup.py`:

```python
# verify_setup.py

import os
from dotenv import load_dotenv
from langchain_openai import ChatOpenAI

# Load environment variables from .env file
load_dotenv()

# 1. Check if the API key is loaded
api_key = os.getenv("OPENAI_API_KEY")
if not api_key:
    print("Error: OPENAI_API_KEY not found.")
    print("Please make sure it's set in your .env file.")
    exit()

print("✅ OpenAI API Key loaded successfully.")

# 2. Initialize the ChatOpenAI model
try:
    model = ChatOpenAI(model="gpt-3.5-turbo")
    print("✅ ChatOpenAI model initialized.")
    
    # 3. Invoke the model with a simple prompt
    print("\nInvoking model...")
    response = model.invoke("Why is the sky blue?")
    
    print("\n✅ Model invoked successfully!")
    print("\nResponse:")
    print("-" * 20)
    print(response.content)
    print("-" * 20)
    
except Exception as e:
    print(f"\nAn error occurred: {e}")

```

Now, run the script from your terminal (make sure your virtual environment is still active):

```bash
python verify_setup.py
```

If everything is configured correctly, you should see a successful output, including a scientific explanation for why the sky is blue!
